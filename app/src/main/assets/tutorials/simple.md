# LineCode 简单模式教程

> **提示**：本地 GGUF 模型推理接口尚未接入，请优先使用 **OpenAI 兼容 / Codex Responses / Anthropic Messages** 协议的云端模型。

LineCode Pro 是一款**在 Android 设备上离线运行的 AI 编程工作台**：选一个本地项目作为工作区，接入大模型，让模型在沙箱里调用文件、Shell、HTTP、Agent 等工具完成真实的代码任务。本教程按 4 步带你完成基础配置。

## 1. 选择工作区（项目）

- 点击聊天顶部**项目名** → 在弹出的"项目"列表里**新建**或**切换**项目。
- LineCode 通过 SAF 访问项目目录；选过的目录会一直保留在列表里。
- 所有文件读写、Shell 命令、Agent 工具调用都会**以当前项目为根目录**，不越界。

> 小贴士：第一次进入 App 时，模型列表为空；先加一个模型，再回来选项目开始对话。

## 2. 添加模型

入口：聊天顶部右上角 **"更多" → 设置 → AI 与模型 → 模型管理 → 添加模型**。

| 模型类型 | 推荐场景 | 注意事项 |
| --- | --- | --- |
| OpenAI 兼容 | 自有 OpenAI / 第三方代理 / DeepSeek / 通义千问等 | 填写 Base URL、API Key、Model ID 即可 |
| Codex Responses | OpenAI 的新 Responses 接口 | 模型 ID 形如 `gpt-4.1` |
| Anthropic Messages | Claude 系列 | 需填 X-Api-Key 头或 `Authorization: Bearer` |
| 本地 GGUF | 当前**不可用** | 推理接口尚未接入，调用会失败，请改用云端协议 |

> **保存后**在"模型管理"列表里**点选**该模型作为当前聊天模型，返回聊天界面即生效。

## 3. 配置工具与执行目标

入口：设置 → **工具与执行**（"Tools & execution"）。

- **执行目标**：可选
  - **本地工作区**：在本机项目里跑文件读写、Shell、HTTP 服务、Agent 等工具。
  - **SSH Shell**：把文件操作、Shell 转发到远端 Linux；本地文件工具会被禁用，Agent、Pipeline、TODO、Web 搜索、图像理解/生成仍可用。
- **权限模式**：
  - **每次询问**：每次工具调用都弹确认卡（默认安全）。
  - **本会话信任**：对当前对话中的同一类工具自动放行。
  - **信任所有**：批量执行时再用，避免误改文件。
- **Web 搜索 / 图像理解 / 图像生成**模型在 **设置 → 工具设置** 里单独选择。

> 推荐入门组合：执行目标 = 本地工作区、权限 = 每次询问；熟悉后再按需放开。

## 4. 开始对话

- 在底部输入框写需求，例如：
  - "阅读 `src/main/java/...` 下的 5 个文件，告诉我这个模块的职责。"
  - "重构 `Foo.kt`，加单元测试。"
  - "用 Web 搜索查一下 Android 16 的新特性并整理成 Markdown。"
- 输入框右侧可以**附加文件 / 目录**作为上下文。
- 工具调用会以**卡片形式**出现在回复中：每个卡片可展开查看参数、运行日志、产物；文件改动会产生 **Diff**，可一键回退。

## 常用功能速查

| 想做什么 | 在哪里 |
| --- | --- |
| 切换 / 新建项目 | 聊天顶部项目名 |
| 看历史对话 | 左侧抽屉 → 对话列表 |
| 压缩上下文（让长对话更省 token） | 右上角"更多" → 压缩上下文 |
| 清空当前对话 | 右上角"更多" → 清空对话 |
| 切换深色 / 浅色 / 咖啡 / 高对比主题 | 设置 → 主题与外观 |
| 调整代码块是否换行、浏览器打开方式 | 设置 → 输出与浏览器 |
| 让 AI 记住项目长期事实 | 设置 → 记忆 |
| 自定义系统提示词、风格 | 设置 → AI 行为 |
| 关闭 / 启用 Web 搜索 API | 设置 → 工具设置 |
| 续命避免后台被杀 | 设置 → 后台保活 |

| 排查问题 / 看看版本号 | 设置 → 关于 |

## 注意事项

- **本地 GGUF 模型推理接口尚未接入**：请优先使用 OpenAI 兼容 / Codex Responses / Anthropic Messages 协议的云端模型；不要选择"Load local model"。
- **网络访问**：所有出站 URL 都会被沙箱策略检查，明文 HTTP 仅允许 `localhost / 127.0.0.1 / 10.0.2.2`。
- **私密数据**：`.linecode` 归档在导出时会**自动脱敏**（API Key、SSH 私钥 / 密码、Web 搜索 Key、自定义 MCP 头），但仍请妥善保管。
- **存储权限**：第一次选项目时系统会要求 SAF 授权；如果失败，可以去 系统设置 → 应用 → LineCode → 存储 重新授权。
- **保活 / 后台运行**：默认不开启"前台服务"以省电；如需长时间挂机任务，进入设置 → 后台保活打开对应选项。
- **遇到崩溃 / Bug**：`设置 → 关于` 查看版本号与日志导出方式，附带日志反馈问题。
