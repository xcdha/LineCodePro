# LineCode 专业模式教程

> **提示**：本地 GGUF 模型推理接口尚未接入，请优先使用 **OpenAI 兼容 / Codex Responses / Anthropic Messages** 协议的云端模型。

这份教程适合已经上手、想"用得更狠"的高级用户——讲的是**怎么高效地让 AI 帮你改代码**，不是这个 App 的源码结构。如果你只想看基本用法，请切到"简单模式"。

## 1. 项目与多工作区

### 多项目并行

- 同一时间只能有一个"当前项目"，但**项目列表**可以保留多个，左侧抽屉 → 项目即可快速切换。
- 切换项目时：所有文件工具、Shell、Agent 的工作目录会**整体迁移**到新项目；未保存的草稿消息不会丢。
- 适合场景：一个项目写业务代码、另一个项目放参考实现、第三个项目做脚本工具。**不要把多个 Git 仓库混在同一个项目里**——后续归档、SSH 转发都会出问题。

### SAF 授权的边界

- 项目根目录一旦选定，**整个子树都可以读**；越界路径（`..`、绝对路径、跳到 `/sdcard/...`）会被沙箱策略拒绝。
- 如果项目里有大量二进制、媒体或 `.git/` 目录：把目录加进"忽略"（暂未提供 UI 白名单，可在提示词里告诉 AI "忽略 `build/` `node_modules/` `*.lock`"）。
- 同一项目重复选择同一目录会被识别为同一个项目，**不会**创建副本。

## 2. 模型选择策略

### 协议对比

| 协议 | 适用模型 | 适合场景 | 注意事项 |
| --- | --- | --- | --- |
| OpenAI 兼容 | 自有 OpenAI、DeepSeek、通义千问、Moonshot、第三方代理 | 默认选项，覆盖最广 | Base URL 必须能直连；公司代理要带 `Authorization` 头 |
| Codex Responses | OpenAI 新 Responses 接口（`gpt-4.1` / `o3` / `o4-mini`） | 需要更稳的工具调用、思考痕迹 | 模型 ID 不要写带日期的快照号 |
| Anthropic Messages | Claude 系列 | 长上下文、深度推理、代码审阅 | 需要 X-Api-Key 或 `Authorization: Bearer` |
| 本地 GGUF | **不可用** | — | 推理接口未接入，调用即失败 |

### 任务—模型匹配

- **短改 / 写函数 / 改单文件**：便宜的小模型（`gpt-4.1-mini` / `claude-haiku-4-5` / `deepseek-chat`），响应快、改动小。
- **跨文件重构 / 大型功能**：旗舰模型（`gpt-4.1` / `claude-sonnet-4-5` / `o3`），需要更强的计划性。
- **代码审阅 / 找 Bug**：Anthropic 系列对 diff 评审体验最好。
- **生成长文 / 文档**：OpenAI 兼容 + 长上下文。
- **多模态**（截图理解）：在"工具设置"里给图像理解单独挑一个支持图片输入的模型，可以和聊天模型不同。

> **省 token 的小技巧**：在模型列表里建好 2-3 个不同档位的模型，需要大动作时切旗舰，跑小循环时切便宜模型。

## 3. 权限模式的取舍

| 模式 | 行为 | 适用场景 |
| --- | --- | --- |
| 每次询问 | 每个工具调用都弹确认卡 | 改生产代码、删文件、改 Git 之前 |
| 本会话信任 | 同一类工具在本对话内自动放行 | 中等风险批改（比如批量重命名） |
| 信任所有 | 一律放行，不弹确认 | 沙箱项目 / 一次性探索 / 大量读操作 |

**实战建议**：
- 默认用"每次询问"，**不要**用"信任所有"——一旦 AI 写错 `rm -rf` 命令，**没有回头路**。
- 遇到一个长任务要连续读 10 次文件，**第一次**确认时勾"本会话信任"，后面就不会再打断。
- "信任所有"只在**只读沙箱**（比如临时导出的演示项目）里用。
- 改完配置想回"每次询问"：切换权限模式立即生效，**不需要重启对话**。

## 4. 工具组合的实战套路

### 套路 A：先读后改

让 AI 先 `list_directory` + `file_read` 看相关文件 → 给出计划 → 你审阅 → 再 `file_edit` / `file_write`。**永远不要**让 AI 一上来就改文件。

### 套路 B：分而治之（Agent 编排）

- 复杂任务拆成子任务，让多个 Agent 各自负责一个模块。
- "**Agent**" 工具：单 Agent 跑一个子任务，**有独立的 system prompt 和工具集**。
- "**Agent Pipeline**" 工具：把多个 Agent **按流水线串起来**，上一个的输出作为下一个的输入。适合"研究 → 设计 → 实现 → 测试"这种链式工作。
- **不要**让一个 Agent 做太多事；超过 5 步的工具链基本会失焦，拆开。

### 套路 C：用 Web 搜索做事实查证

- AI 写代码时遇到不确定的 API、版本、文档，让它先 `web_search` 再下笔。
- 在"工具设置 → Web 搜索"里填 API Key（默认未开启）。
- 提示词示例："先 `web_search` 查 Android 16 的新 API，再基于结果改代码。"

### 套路 D：图像理解 + 图像生成

- **图像理解**（截图提问）：把截图附在消息里，AI 调 `image_understanding` 自动解析。
- **图像生成**（生成 UI mockup）：让 AI 直接 `image_generation`，会插入图片到对话。
- **两个模型可以独立配置**，不必是同一个。

### 套路 E：TODO 工具追踪进度

- 长任务（> 10 步工具调用）让 AI 用 `todo_update` 维护任务清单；进度可见、可中断后继续。

## 5. 自定义扩展

### 自定义 Agent（`agentx_*`）

- 入口：设置 → 扩展 → Agent → 新建。
- 一个 Agent = 一组**预定义工具 + 专属 system prompt + 用户提示模板**。
- 用例：写一个 "Kotlin 重构 Agent"，只允许 `file_read` / `file_edit` / `shell_execute`，system prompt 固定为"Kotlin 代码风格指南"。
- 提示：可以**用 AI 生成草稿**——点"用模型生成"，输入一段自然语言描述，AI 会自动生成工具列表 + 提示词；再人工微调。

### 自定义 MCP（`mcpx_*`）

- 入口：设置 → 扩展 → MCP → 新建。
- 通过 HTTP 接入外部 MCP 网关（符合 MCP 协议的 HTTP 端点）。
- 适用：接入公司内部的代码搜索服务、自定义部署的 SQL 工具、内部 API 网关。
- 保存后即可在工具列表里看到并调用。

### Skills

- 入口：项目根或全局 `Skills/` 目录 → 放 `SKILL.md`（或 `.zip` / 目录）。
- Skill 是**给 AI 看的"领域知识包"**——比如项目特定的命名规范、CI 命令、部署流程。
- 例子：在项目根创建 `Skills/deploy.md`，写明"部署前先跑 `npm run build`，再 ssh 到 prod-1 跑 `pm2 reload app`"，AI 会在相关对话里自动引用。

## 6. 上下文管理

### 什么时候该"压缩上下文"

- 聊天顶部出现"上下文紧张"提示时，**立刻**手动触发"更多 → 压缩上下文"。
- 长对话（> 30 条消息）每完成一个里程碑**主动**压缩一次。
- 切换到便宜模型做下一阶段任务**之前**，压缩一次，省 token。

### 记忆的用法

- 入口：设置 → 记忆。
- **长期事实**（项目名、技术栈、约定）→ 长期记忆。
- **当前任务上下文**（"正在做 XX 功能"）→ 工作记忆。
- **临时知识**（"本次对话里发现 XXX"）→ 短期记忆。
- **不要**把敏感凭据（API Key、密码）写进记忆——记忆会被导出到 `.linecode` 归档。

### 工具结果过长

- AI 一次读了 10 个文件 / 一个 5000 行的日志：上下文会被迅速吃光。
- 提示词加一句："**只返回关键部分**"或"先 `head -100`，再让我看更多"。
- 必要时在提示词里指定"用 `glob` 找文件名，再挑相关的读"。

## 7. SSH / Termux 集成

### SSH 远程工作流

- 入口：设置 → SSH 连接 → 填主机/端口/用户名/密码或私钥。
- 切换"执行目标"为 **SSH Shell** 后：所有文件工具 + Shell 都在远端 Linux 执行；本机项目路径**不会**生效。
- 适用：CI 调试、远端构建（Android 在远端编译更稳）、团队共享沙箱。
- **私钥安全**：私钥在 App 内加密保存，但导出 `.linecode` 归档时会被**自动脱敏**（远端主机只有"导入新设备后重填"一种同步方式）。

### Termux 集成

- 入口：设置 → 工具与执行 → Termux 集成。
- 自动通过 SAF 写入 `assets/termux_setup.sh` 到 Termux，建立 sshd（端口 8022）。
- 之后可以填 "主机 = `localhost`、端口 = 8022"走 SSH 协议，但执行的是本机的 Termux 环境。
- 优势：Termux 自带 `proot-distro`，能跑完整 Linux 工具链，**比 Android 直接执行 Shell 强很多**。

## 8. 提示词技巧

### 让 AI 听话的句式

- **明确范围**："只改 `src/api/` 下的 3 个文件，不要动其他文件。"
- **明确输出格式**："最后给我一个 Markdown 总结：改动列表 / 关键 diff / 测试建议。"
- **明确不要做的事**："不要加新依赖。""不要重命名现有方法。"
- **明确验收标准**："完成后跑 `npm test`，把输出贴回来。"

### 分轮迭代

- **第一轮**：让 AI 列出"我打算改什么"，**不**实际改。
- **第二轮**：你审阅计划，**再**让 AI 改。
- **第三轮**：让 AI 跑测试 / 编译，**回报**错误。
- 一次性让 AI 改+跑+修错，**容易**跑飞。

### 工具调用的边界

- 想让 AI **必须**读某文件：在消息里直接拖进去做附件。
- 想让 AI **不要**读：写明"文件已附在消息中，不需要再 read"。
- 想强制某工具：写"用 `file_edit` 而不是 `file_write` 改"——避免覆盖整个文件。

## 9. 排查清单

| 现象 | 排查路径 |
| --- | --- |
| 工具调用被拒 | 权限模式 / 当前项目的 SAF 授权 / 文件路径越界 |
| 模型一直报超时 | 切换更快的模型 / 改小 base URL 的代理 / 检查网络 |
| GGUF 模型一调用就失败 | **已知限制**，本地 GGUF 推理未接入，请改用云端协议 |
| 工具卡片显示"执行失败" | 长按卡片看完整堆栈；常见：权限不足、命令找不到、超时 |
| 上下文紧张 | 更多 → 压缩上下文；或开启"AI 行为 → 自动压缩" |
| 导入 `.linecode` 后模型列表为空 | 预期行为：模型 API Key 因安全脱敏被清空，需重新填写 |
| Web 搜索没结果 | 检查"工具设置 → Web 搜索"是否配置了 API Key；不少搜索服务已不再免 |
| 文件改动看不到 Diff | 在工具调用卡片右上角点"展开" → 切到 Diff 视图 |
| SSH 连接失败 | 设置 → SSH → "测试连接"；私钥格式不对是高频原因 |
| Termux 集成为空白 | Termux 没装 `openssh`；按提示在 Termux 里跑 `pkg install openssh` |

## 10. 性能与电量

- **后台保活**：默认关闭。长任务（> 30 分钟）才需要开；开了会显著耗电。
- **WakeLock**：同上，只在挂机任务时开。
- **聊天模型选择**：便宜模型响应快、电量友好；旗舰模型耗电高 3-5 倍。
- **流式响应**：默认开，体验更好但费电；电池紧张时可以关。
- **Markdown 渲染**：长消息在低内存设备上略卡；切"代码自动换行"会重排，频繁切换会卡顿。

## 11. 数据管理与协作

### 导出 `.linecode` 归档

- 设置 → 数据管理 → 导出：包含全部对话、模型（**API Key 脱敏**）、SSH 凭据（**密码 / 私钥 / 密码短语 脱敏**）、Web 搜索 Key、扩展、记忆、主题、提示词。
- 文件名形如 `LineCode Pro <versionName>.linecode`，**自带版本号**——不同版本之间导入可能不兼容。
- 适合：换机、备份、给同事分享配置（**不**适合分享凭据——脱敏后对方必须重填）。

### 导入

- 设置 → 数据管理 → 导入：选 `.linecode` 文件 → 弹"确认导入"对话框。
- 导入后**一定会缺失**：模型 API Key、SSH 凭据、Web 搜索 Key、自定义 MCP 的敏感头——这些都是脱敏项。
- 提示词模板、主题、记忆、扩展 Agent / MCP 草稿会**正常**导入。

### 团队协作建议

- 团队成员**各自**维护自己的模型 / SSH 凭据；归档里只放公共部分（提示词、Skills、扩展草稿）。
- 共享的 Skills 放进项目根 `Skills/`，**跟随 Git**——比走 `.linecode` 归档分发更高效。

## 12. 主题与外观

- 设置 → 主题与外观：跟随系统 / 浅色 / 深色 / 咖啡纸 / 高对比。
- 自定义颜色：5 种主色（背景 / 表面 / 边框 / 强调 / 文本），保存后立即生效。
- 主题切换**立即生效**，不需要重启 App。
- 深色模式在 OLED 屏上更省电；咖啡纸适合长时间阅读代码。
- 字体大小：跟随系统；在无障碍设置里调大系统字号即可全局生效。
